# 05｜一张五层参考架构：Interface / Capability / Data&Memory / Runtime / Governance

> 适合发在知乎：架构师友好、易收藏。

当企业希望 Agent 从“局部助手”变成“可持续运行的执行网络”，系统组织方式会从“应用分层”转向“能力分层”。

下面给出一个可落地的 **五层参考架构**（职责边界比技术选型更重要）。

---

## 五层参考架构（职责版）

1) **Interface Layer（统一接入面）**
- API Gateway、Discovery、Schema 注册、鉴权、限流
- 解决：Agent 如何稳定连上企业世界

2) **Capability Layer（能力编排层）**
- 能力目录/能力图、策略绑定、版本与生命周期
- 解决：连上之后能不能“正确完成任务”

3) **Data & Knowledge + Memory Layer（上下文底座）**
- 结构化事实、制度/SOP 知识、任务/经验记忆
- 解决：Agent 凭什么持续做出正确判断

4) **Runtime & Orchestration Layer（执行控制层）**
- 任务状态机、调度、重试/补偿、人工接管、成本控制
- 解决：从生成到交付

5) **Security & Governance Layer（安全与治理层）**
- 身份映射、细粒度授权、策略引擎、审计还原、合规留痕
- 解决：能不能放心上线

---

## 你可以用它做什么

- 对齐跨团队讨论：每个人先在同一张“职责地图”上讲话
- 做架构评审：评审“缺哪层/哪层薄弱”，而不是只争模型/框架
- 做路线图：先补最短板，再扩大自动化边界
